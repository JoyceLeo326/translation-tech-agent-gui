# A组第二次提交独立复核摘要

## 结论

A 组本次更新补交了包含真实 SVG 的 Word，证明其样本中的 SVG 文字以 `<text>/<tspan>` 节点存在；但原包不能直接验收或上传 GitHub。独立复核已完成可处理问题的修正，并生成可复现的完整修正版。

## 原提交问题

1. `S2_translate.py` 包含真实翻译服务账号和密钥，存在凭据泄露风险。
2. `all_pictures` 有 39 个文件，而当前 Word 只有 31 个媒体；`image8.jpeg`、`image24.png` 等同名文件与 Word 内媒体不一致。
3. SVG 回填采用全文字符串替换，不限定 XML 文字节点，也没有对文字路径化的 SVG 给出明确处理策略。
4. 清单有 81 条机器译文，但没有复核人、日期或确认列；多处译文不准确。
5. OCR 把第六章封面的装饰圆形识别为字母 `O`，并漏掉多处显著文字。
6. 原回填结果存在大面积白块、字号过小和中文未覆盖等问题。

## 已完成修正

- 删除在线翻译依赖和所有凭据，使用审校后的确定性翻译清单。
- 以源 Word 为唯一媒体来源，从干净目录重新提取，避免残留图片错配。
- 复核 71 个实际使用的图中文字项，修正 OCR 错字、机器译文和漏译。
- 对 10 个真实 SVG 按 `<text>/<tspan>` 节点翻译，并调整英文文字字号。
- 对无可编辑文字但含路径的 SVG 给出明确的“需要渲染和 OCR”错误。
- 重新排版 20 个有文字的栅格媒体；二维码保持原字节不变。
- 生成最终 Word、审校清单、译后媒体、预览图、验证 JSON 和 PDF。

## 验证结果

- Python 语法检查通过。
- 源 Word 与最终 Word 均通过 ZIP 完整性检查。
- 最终 Word：31 个媒体、10 个 SVG、31 个图片关系、32 个嵌入引用。
- 30 个媒体发生预期变化，二维码媒体 `image1.jpeg` 与源文件字节完全一致。
- Microsoft Word 16.0 成功导出 17 页 PDF，逐页预览未发现空白、媒体缺失或页面溢出。
- 修正版目录未包含原提交的凭据值。

## 仍需 A 组本人完成

以下事项无法由独立复核代替：

1. 立即在翻译服务后台吊销并轮换已经泄露的密钥，并确认旧密钥失效。
2. 由 A 组真实成员对 71 项审校清单署名、填写日期并承担人工终审责任。
3. 提供一个文字已经路径化的真实业务 SVG 样例和运行记录。修正版已通过受控负向测试，但 A 组本次只提供了可编辑文字节点型 SVG。
