# A组整改需求

A组同学：

你们提交的工作需要按下面要求补齐和整改。请注意，本次要求不是“能跑一下”即可，而是要做到结果真实、完整、可复现、可验收。

## 一、必须解决的问题

1. 不允许跳过、删除或忽略 SVG/矢量图。
   - 如果 Word 原稿中确实存在 `.svg`，必须自动处理。
   - 对包含 `<text>` / `<tspan>` 文本节点的 SVG，应直接解析 XML、提取中文、翻译后原位写回。
   - 对已经转成路径、没有可编辑文字节点的矢量图，必须说明原因，并采用“渲染为图片 + OCR + 回填”的方案处理，不能直接删除。

2. 必须保证文档图片数量完整。
   - 原文档、PDF 或资源包中出现的所有插图都要进入最终结果。
   - 如果发现提交版 Word 中已经缺图，必须从 PDF/原始材料中恢复，并在日志里写清楚来源、页码、文件名。

3. OCR 和翻译结果必须人工核对。
   - 不得直接使用明显错误的 OCR/机翻结果。
   - 专名、术语、成语、文化表达必须校正，例如“徽派头墙”“客家土楼”“四大发明”等不能误译。
   - 二维码、装饰性小字、无须翻译的元素不得被误覆盖。

4. 图片回填必须美观且不遮挡关键内容。
   - 原中文应被完整覆盖，不允许中英文字重叠。
   - 英文要在原文字区域内完整显示，不得超框、截断、压到图像主体。
   - 黑白图、彩色图都要分别检查。

5. 脚本必须可复现。
   - 提供一键运行脚本。
   - 不得硬编码个人 API Key、密钥或账号信息，统一从环境变量或配置文件读取。
   - 路径不能写死为某台电脑独有路径。
   - 运行后应能生成：提取图片、OCR/翻译清单、回填图片、最终 Word。

## 二、必须提交的材料

1. 最终 Word 文档。
2. 完整 Python 脚本。
3. OCR/翻译结果表格，至少包含：图片名、原文、译文、坐标或位置说明、是否人工校正。
4. 回填后的全部图片文件夹。
5. 日志文档，说明：
   - 原始 Word 中有哪些媒体文件；
   - 是否存在 SVG；
   - SVG 如何处理；
   - 是否有缺失图片；
   - 缺失图片从哪里恢复；
   - 最终媒体数量和图片引用数量。
6. 图片总览图或截图，方便快速检查所有图片是否处理完整。

## 三、验收标准

1. 最终 Word 能正常打开，无损坏、无丢图。
2. 文档中的所有图片都已处理，不能少图。
3. 若原始材料有 SVG/矢量图，最终结果中必须有对应处理结果。
4. 脚本能在干净目录中按说明重新跑出结果。
5. 不存在硬编码密钥。
6. 图片译文准确、自然、与中国文化主题相符。
7. 随机抽查任意图片时，不能出现中文残留、英文截断、覆盖错位、误删二维码等问题。

## 四、特别说明

不要用“电脑环境不行”“AI没解决”“手动转一下再处理”作为最终方案。可以说明环境问题，但必须给出自动化替代路径和可运行实现。

如果某类图确实无法从当前材料中恢复，必须提供证据，例如 Word 包内 `word/media` 列表、PDF 页码截图、脚本检测结果等。不能凭猜测说“没有”或直接跳过。
