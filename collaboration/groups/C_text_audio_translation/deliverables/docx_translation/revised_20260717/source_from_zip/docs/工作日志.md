# DOCX翻译工作流 - 工作日志与交付文档

## 一、项目概述

- **项目名称**：DOCX双支路翻译工作流
- **功能描述**：将中文DOCX文档翻译为英文DOCX，保留原始格式（字体、加粗、斜体、表格、页眉页脚等）
- **工作流架构**：双支路设计
  - **支路1**：DOCX → 解析 → 分片 → 翻译 → 三列对照Excel（中文原文 | 机器英文译文 | 人工审核）
  - **支路2**：审核后的Excel + 原始DOCX → 英文DOCX（替换文本，保留格式）

## 三、支路二使用方式确认

### 3.1 当前实现：**方案A（同时上传）**

支路2要求用户**同时上传**两个文件：
- `file_docx`：原始DOCX文档（必填）
- `review_excel`：人工审核后的Excel（必填）

### 3.2 输入变量定义

```python
class GraphInput(BaseModel):
    file_docx: Optional[File] = Field(default=None, description="待翻译的DOCX文档（支路1必填，支路2必填，作为回填原始DOCX）")
    review_excel: Optional[File] = Field(default=None, description="人工审核后的Excel（支路2必填）")
```

### 3.3 路由逻辑

```python
def router_node(state):
    if state.review_excel is not None:
        return "支路2"  # 生成英文DOCX
    elif state.file_docx is not None:
        return "支路1"  # 生成三列对照Excel
```

### 3.4 为什么支路2必须同时上传DOCX

- **避免全局状态依赖**：每次test_run是独立的，全局状态不保留
- **确保DOCX匹配**：审核后的Excel必须与DOCX内容匹配
- **防止误替换样例文件**：缺少原始DOCX时直接报错，不允许回退到样例URL

## 六、验收标准

### 6.1 行数对齐

- **标准**：Excel行数 = DOCX解析后的句子数
- **允许例外**：标题行（不按句子拆分，独立成行）

### 6.2 中文残留

- **标准**：英文DOCX中**完全0中文残留**
- **允许例外**：
  - 页眉页脚中的非文本内容（如页码字段）
  - 图片中的文字（无法替换）
  - 文本框中的特殊字符

### 6.3 格式保留

- **必须保留**：
  - 段落样式（字体、字号、颜色、加粗、斜体、下划线）
  - 表格结构和单元格样式
  - 页眉页脚文本
  - 图片位置和内容
- **允许例外**：
  - 文本框（当前未处理）
  - 批注（当前未处理）
  - 脚注（当前未处理）

### 6.4 文件命名

- **标准**：由LLM自动翻译为英文
- **示例**：`童话故事.docx` → `Fairy Tales.docx`
- **允许例外**：翻译失败时使用原始文件名

### 6.5 失败处理

- **标准**：支路2失败时**中断**，不允许部分输出
- **原因**：部分替换会导致DOCX内容不完整

## 八、技术实现要点

### 8.1 DOCX解析

- **遍历方式**：XML body顺序遍历（`<w:p>`段落 + `<w:tbl>`表格）
- **标题识别**：基于内容特征（短段落<30字符 + 无句号/逗号）
- **句子拆分**：按句号/问号/感叹号拆分（`。！？`）
- **页眉页脚**：提取时添加`[页眉]`/`[页脚]`标记，替换时自动去除

### 8.2 文本替换

- **替换方式**：直接操作XML（ZIP内修改`<w:t>`文本节点）
- **格式保留**：按比例分配英文到各run，保留字体/加粗/斜体
- **反向匹配**：段落是键值的子串时也匹配（处理合并文本）
- **normalize_text**：去除空格、统一标点，消除文本差异

### 8.3 页眉页脚处理

- **提取**：遍历`word/header*.xml`和`word/footer*.xml`
- **页码检测**：检测`<w:fldChar>`字段，跳过页码数字
- **替换**：只替换非页码的`<w:t>`元素，页码保持原样

## 九、使用流程

### 9.1 支路1（DOCX → Excel）

1. 上传DOCX文档（`file_docx`）
2. 工作流自动解析、翻译
3. 输出三列对照Excel（`origin_excel`）
4. 用户下载Excel，人工审核"人工审核"列

### 9.2 支路2（Excel → 英文DOCX）

1. 上传审核后的Excel（`review_excel`）
2. **必须同时上传原始DOCX**（`file_docx`）
3. 工作流自动替换文本
4. 输出英文DOCX（`output_en_docx`）和替换覆盖报告（`docx_replace_report`）
