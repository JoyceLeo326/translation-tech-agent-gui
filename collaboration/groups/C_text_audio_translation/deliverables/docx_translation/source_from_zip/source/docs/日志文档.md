# 日志文档

## 目的
搭建一个双支路DOCX翻译工作流，实现：
- **支路1**：上传中文DOCX → 自动解析层级（标题/正文/表格）→ 分批翻译 → 输出三列对照Excel（中文原文 / 机器英文译文 / 人工审核）
- **支路2**：上传校对完的Excel → 读取第三列人工审核内容 → 回填替换原始DOCX的中文文本（保留原图、表格、排版、字体格式）→ 输出英文DOCX

## 动作
1. 使用LangGraph 1.0搭建有向无环图工作流，含条件路由、循环子图
2. 实现DOCX解析节点：XML body顺序遍历段落+表格，按句号拆分为单句，合并断行中文文本
3. 实现分批翻译节点：1行/批，带行号对齐确保不丢不漏，LLM输出行数不足时用原文填充
4. 实现Excel生成节点：输出三列对照表，上传到对象存储
5. 实现Excel读取节点：读取第三列"人工审核"内容，生成中文→英文键值对
6. 实现DOCX回填节点：直接操作ZIP内XML（替换`<w:t>`文本节点），按比例将英文分配到各run，保留每个run的字体/加粗/斜体等格式
7. 输出文件名自动翻译：提取原始DOCX文件名 → LLM翻译为英文 → 用英文名输出

## 工作流节点清单

| 节点名 | 类型 | 功能 | 所属支路 |
|-------|------|------|---------|
| router_node | 路由 | 判断输入参数，决定走支路1还是支路2 | 公共 |
| docx_parse_node | 任务 | 解析DOCX，提取段落和表格文本，按句号拆分，合并断行 | 支路1 |
| split_node | 任务 | 将文本按1行/批分片 | 支路1 |
| translate_batch_node | 循环子图 | 分批调用LLM翻译，带行号对齐 | 支路1 |
| merge_node | 任务 | 合并翻译结果 | 支路1 |
| generate_excel_node | 任务 | 生成三列对照Excel并上传对象存储 | 支路1 |
| read_excel_node | 任务 | 读取Excel第三列，生成中文→英文翻译映射 | 支路2 |
| generate_docx_node | 任务 | 直接操作XML回填英文，保留所有run格式，输出英文DOCX | 支路2 |

## 输入变量

| 变量名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| file_docx | File | 是 | 上传的DOCX文件（走支路1），或原始DOCX文件（走支路2） |
| review_excel | File | 否 | 已校对完成的Excel文件（走支路2），须包含3列：中文原文 / 机器英文译文 / 人工审核 |

## 输出变量

| 变量名 | 类型 | 说明 |
|-------|------|------|
| output_excel | File | 支路1输出：三列对照Excel（中文原文 / 机器英文译文 / 人工审核列） |
| output_en_docx | File | 支路2输出：英文版DOCX（文件名由原始中文名翻译为英文） |

## 产出
- **工作流源码**：`src/graphs/` 目录下9个节点文件 + 状态定义 + 图编排 + 循环子图
- **LLM配置**：`config/` 目录下2个模型配置文件（翻译 + 文件名翻译）
- **测试结果**：
  - 支路1：133行Excel，100%英文覆盖，0重复
  - 支路2：78段落全部替换为英文，0中文残留，格式保留
- **最终输出文件**：原始DOCX中文文件名翻译为英文后作为输出名（如"中国文化知识百科.docx" → "Encyclopedia_Of_Chinese_Culture.docx"）

## 需求
- 后续上传新的DOCX时，先用支路1生成对照Excel，人工校对后走支路2生成英文DOCX
- 输出DOCX文件名固定规则：原始文件名翻译为英文后作为输出名
- 工作流以文件输入为主，支持DOCX和Excel两种文件类型
