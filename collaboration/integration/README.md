# 总整合区

这里是最终 GUI、智能体和工作流整合时使用的管理区。

## 子目录

- `manifests/`：各组交付清单、整合状态、字段规范。
- `adapters/`：A/B/C 组接入入口、实现位置和读取范围说明。
- `final_outputs/`：最终提交或演示用输出。

## 整合原则

1. 优先读取各组 `deliverables/` 下的稳定成果。
2. 跨组公共依赖从 `shared/` 读取，例如术语库和审校表。
3. 不直接依赖组员个人临时目录或仓库外本地路径。
4. 每次接入一个组的成果后，更新 `manifests/group_deliveries.md`。
5. 同组存在多次提交时按日期和验收记录选取最新版；旧版只作历史留存，不再作为 GUI 和打包快照的默认输入。
